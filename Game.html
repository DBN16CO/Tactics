Game.routeunithelper module¶

This file will handle all routed methods managing unit data All methods must follow the following standards:

Inputs - they will all take in only one input: the JSON data

Outputs - they will pass back a formatted JSON response object which will detail the success or failure of the command as well as any other necessary information regarding the command.

Game.routeunithelper.setTeam(data)¶

Called when a user wants to set their team lineup before starting a game. Will store their unit selection as well as their leader, his ability, and the perk data Command: ST (Set team)

Parameters:data (Dictionary) –

The necessary input information to process the command, should be of the following format:

{

“Leader”:”Sniper”,

“Ability”:”Increased Range”

“Units”:[“Archer”,”Archer”,”Mage”,”Flier”,”Swordsman”,”Swordsman”,”Swordsman”,”Swordsman”],

“Perks”:[“Extra Money”,”Strong Arrows”,”Mountain Fighter”]

}

Notes:

  • All of the values provided for each key are the X{Name} chosen for that key.
  • If no perks are chosen an empty list should still still be sent for that key
Return type:Dictionary
Returns:A JSON object noting the success of the method call:

If Successful:

{“Success”:True}

If Unsuccessful:

{“Successful”:False,
“Error”:”You did not provide the necessary information.”}

Notes:

  • The error message provided should be of an acceptable form such that errors can be directly displayed for the user.
Game.routeunithelper.takeAction(data)¶

Called when a user wants to take an action with a specific unit, such as attacking an opponent, healing an ally, or simply moving to a new location. Command: TA (Take Action)

Parameters:data (Dictionary) –

The necessary input information to process the command, should be of the following format:

{

“Action”:”Attack”,

“Game”:”vs. opponent #1”,

“Unit”:1,

“X”:3,

“Y”:4,

“Target”:9

}

Notes:

  • Action is the NAME of a valid action
  • The value for Unit and Target are the unit’s ID
  • The X and Y values are the new location for the unit
  • Depending on the action, the Target may be optional
  • Both X AND Y are required
Return type:Dictionary
Returns:A JSON object noting the success of the method call:

If Successful:

{

“Success”:True

“Unit”{

“ID”:1,

“HP”:10,

“NewHP”:10

},

“Target”:{

“ID”:9,

“HP”:10,

“NewHP”:0

}

}

Notes:

  • The “Target” object could be omitted if the action did not involve a target.
  • The HP for the unit could be omitted if the action is “Wait” or “Heal”
  • The HP is the previous HP

If Unsuccessful:

{“Successful”:False,
“Error”:”You did not provide the necessary information.”}

Notes:

  • The error message provided should be of an acceptable form such that errors can be directly displayed for the user.

Game.unithelper.saveActionResults(action, game, unit_dict, target_dict=None)¶

Updates the game with the provided action

Parameters:
  • action (Action object) – The action taken
  • game (Game object) – The game being updated
  • unit_dict (Dictionary) –

    Necessary components to describe the updated unit, should contain:

    • “Unit” which is the unit object
    • “NewX” which is the X coordinate to which the unit is moving
    • “NewY” which is the Y coordinate to which the unit is moving
    • “HP” if a target was specified, which is the old HP for this unit before the action
    • “NewHP” if a target was specified, which is the new HP for this unit after the action
  • target_dict (Dictionary) – Necessary components to describe the targeted unit, should have “Unit” and “HP” keys which act the same as those in the unit_dict
Return type:

Boolean
Returns:

True if the update was successful, False otherwise
